# Architecture — Voice of Gen 2026

**Live:** `giftforaleyna.arfannazar.com`
**Repo:** `github.com/arfannazar-ai/voice-of-gen-2026`
**Stack:** Next.js 16 (App Router) · TypeScript · Framer Motion · Anthropic SDK · Vercel

---

## What This App Does

A single-user AI communication tool built for Aleyna Sayın, AIESEC leader. She answers 5 questions to set context, dumps raw thoughts into a textarea, and receives a polished AIESEC communication piece generated by Claude Sonnet 4.6 — streamed live. Past outputs are archived in a sidebar ("Love Letters"). She can copy and open LinkedIn in one click.

## Data Flow

```
User fills Questionnaire (5 steps)
        ↓
QuestionnaireData stored in page.tsx state
        ↓
MainApp renders with that context
        ↓
User types raw thought → clicks "Speak Up"
        ↓
ThoughtDump POSTs to /api/generate
        ↓
route.ts calls client.messages.stream() via Anthropic SDK
        ↓
ReadableStream piped back to client
        ↓
ThoughtDump reads chunks via TextDecoder, renders live
        ↓
onGenerate(thought, output) → new LoveLetter added to state
        ↓
Sidebar shows archived letter, CopyLinkedInButton appears
```

**All state is ephemeral (React useState).** No database. Refresh = everything gone.

## CSS Architecture Note

**Tailwind PostCSS was intentionally removed.** The build environment (macOS, Turbopack, Node at `/opt/homebrew/bin/`) could not resolve `node` in the default PATH when Turbopack spawned child processes for `@tailwindcss/postcss`. Rather than requiring system-level PATH changes (needed `sudo`), all Tailwind utility classes were rewritten as explicit CSS rules in `globals.css` (~230 lines). This is stable but has a maintenance implication: any new utility class used in JSX must be manually added to `globals.css`.

## Technical Debt

### High Priority

**1. No persistence — session data is lost on refresh**
`LoveLetter[]` and `QuestionnaireData` live only in React state. A page reload destroys all history. Fix: `localStorage` for quick win, or a proper DB (Supabase/Neon) for multi-session persistence.

**2. No authentication**
The app is publicly accessible at the domain. Anyone who has the URL can use it and consume API credits. Fix: add a simple passphrase gate (env var checked on the API route) or NextAuth.

**3. API key exposed through chat tooling during setup**
The Anthropic API key was written to `.env.local` during setup and was visible in system tooling. The key should be rotated on `console.anthropic.com` and a fresh key set on Vercel (Settings → Environment Variables). The `.env.local` key is only used locally and is gitignored.

**4. No rate limiting on `/api/generate`**
Any client can POST to `/api/generate` unlimited times. Each call costs Anthropic credits. Fix: add IP-based rate limiting (e.g. Vercel's `@vercel/kv` + a simple counter) or a request token.

**5. CSS utility gap risk**
If a developer adds a new Tailwind class in JSX without adding the corresponding rule to `globals.css`, it will silently fail to render. There is no build-time warning. Fix: either restore Tailwind (requires resolving the PostCSS/PATH issue with a proper Node symlink or Docker build), or establish a convention of always checking `globals.css` when adding classes.

### Medium Priority

**6. `QuestionnaireData.mood` is typed as `LeadershipMood | null`**
The questionnaire always sets mood before proceeding, so `null` is never actually reached in practice. However, downstream components cast it with `as LeadershipMood`. A proper fix would narrow the type after step 1 completes so downstream code doesn't need the cast.

**7. `LoveLetter.id` uses `Date.now().toString()`**
Not collision-proof if multiple letters are generated in the same millisecond. Low risk in practice for a single-user app, but should use `crypto.randomUUID()` for correctness.

**8. Error boundary missing**
There is no React error boundary. An unhandled error in any component will crash the entire app with a blank screen. Fix: wrap `MainApp` and `ThoughtDump` in an `<ErrorBoundary>`.

**9. Streaming error handling is partial**
If the stream starts successfully but fails mid-way (e.g. network drop after headers are sent), the client currently receives a truncated output with no error indicator. The `done` state is only set at the end of the stream loop, so partial outputs render silently as complete.

### Low Priority

**10. `MOOD_COLORS` is defined in three separate files**
`MainApp.tsx`, `Sidebar.tsx`, and `ThoughtDump.tsx` each define their own `MOOD_COLORS` record. Should be extracted to `app/lib/constants.ts`.

**11. `CopyLinkedInButton` hardcodes `linkedin.com/feed/`**
The `linkedInUrl` prop defaults to LinkedIn's feed rather than a new post composer. LinkedIn's post composer URL (`https://www.linkedin.com/feed/?shareActive=true`) would be more useful, but the clipboard-then-navigate flow already achieves this functionally.

## For the Handoff Engineer

Areas that need the most attention if this is being developed further:

1. **Persistence** — the single biggest UX gap. Implement `localStorage` first (30 min) to survive page refreshes, then consider a proper DB if multi-device sync is needed.
2. **Auth gate** — before sharing widely. A single env-var passphrase check in `route.ts` is sufficient for now.
3. **CSS system** — decide: either fix Tailwind PostCSS (requires system Node symlink or Docker) or commit to the manual CSS approach and document the convention clearly.
4. **API key rotation** — the key used during initial setup should be rotated. See Technical Debt item #3.

## Next 3 Logical Steps to Complete the MVP

### Step 1 — Persistence (localStorage)
Store `LoveLetter[]` in `localStorage` so the history survives page refreshes. This is a 30-minute change in `MainApp.tsx`:
```ts
// Load on mount
const [letters, setLetters] = useState<LoveLetter[]>(() => {
  const saved = localStorage.getItem('love-letters');
  return saved ? JSON.parse(saved) : [];
});
// Save on change
useEffect(() => {
  localStorage.setItem('love-letters', JSON.stringify(letters));
}, [letters]);
```
Add a "Clear history" button in the sidebar footer.

### Step 2 — Access Gate
Add a simple passphrase to prevent public access and API credit drain. In `route.ts`, check a `PASSPHRASE` header against `process.env.ACCESS_PASSPHRASE`. In the UI, show a one-time passphrase input on first load (stored in `localStorage`). No auth library needed.

### Step 3 — "Refine This" Loop
After a Love Letter is generated, add a **"Refine"** input below the output that lets Aleyna say what she wants changed (e.g. "make it shorter", "less formal"). This sends the original output + the refinement note back to `/api/generate` as a follow-up message, enabling a real editing loop instead of starting from scratch each time.
